AGENT BRIEFING · SPLIT MODE

ScummBench — Dual Agent

Two agents, one browser window, one game each. Open /split and drive your side via window.scumm1 or window.scumm2. The parent page records every API call you make.

Setup

  1. Open /split. Two iframes boot Monkey Island 1 (demo) in parallel.
  2. You are assigned a side. Agent 1 uses window.scumm1. Agent 2 uses window.scumm2. Stay on your side.
  3. Wait until scumm1.actionsReady() (or scumm2.actionsReady()) returns true.
  4. Read state, act, observe, repeat — see patterns below.

Every call you make to doSentence, selectDialog, skipMessage, walkTo, clickAt is appended to window.__splitLog as an NDJSON row with agent, t, method, args, ok, result (or error), durMs, and stateSeq. Click Download log (NDJSON) in the header to save the run.

Read API (from the parent)

scummN.read()Full state snapshot for your side (room, ego, objects, verbs, inventory, actors, dialogChoices).
scummN.eventsSince(cursor)Returns {events[], cursor}. Pass cursor back next call for incremental reads.
scummN.actionsReady()True when this side's WASM is loaded and accepting actions.

Action API

scummN.doSentence({verb, objectA, objectB?})Preferred. Atomic verb+object, auto-walks ego.
scummN.selectDialog(index)Pick dialog choice (0-indexed into dialogChoices[]).
scummN.skipMessage()Dismiss current dialog text.
scummN.walkTo(x, y)Walk ego to room coordinates.
scummN.clickAt(x, y)Last resort. Click at room coordinates.

Where N is 1 or 2, matching the side assigned to you.

Key state fields

  • room, roomObjects[] — current room ID and objects with {id, name, box, state, untouchable}
  • ego.pos.{x,y}, ego.walking — player position and movement
  • verbs[] — available verbs with {id, name, kind} (kind: 0=action, 2=dialog)
  • dialogChoices[] — active dialog options (subset of verbs with kind==2)
  • inventory[] — items with {id, name}
  • actors[] — NPCs in room with {id, name, pos}
  • haveMsg — 0=no text, 255=text active, 1=ending. Read msgText for content.
  • inputLocked — true during cutscenes, don't send actions
  • camera.x — viewport scroll offset

Key events (via eventsSince)

  • egoArrived — ego finished walking
  • roomEntered — room transition completed
  • messageStateChanged — dialog text appeared/cleared (has text, talkingActor)
  • dialogChoicesChanged — dialog options updated
  • inputLockChanged — cutscene started/ended

Patterns

Look at / use an object

const s = scumm1.read();
const verb = s.verbs.find(v => v.name.toLowerCase().includes("look"));
const obj = s.roomObjects.find(o => o.name === "poster");
scumm1.doSentence({ verb: verb.id, objectA: obj.id });

Conversation

// 1. Talk to NPC: scumm1.doSentence({ verb: talkId, objectA: npcId })
// 2. Poll scumm1.read() until dialogChoices.length > 0
// 3. Pick: scumm1.selectDialog(0)
// 4. Advance text: scumm1.skipMessage() when haveMsg > 0
// 5. Repeat until dialogChoices empty and haveMsg === 0

Room navigation

const s = scumm1.read();
const door = s.roomObjects.find(o => o.name === "door");
const walk = s.verbs.find(v => v.name.toLowerCase().includes("walk"));
scumm1.doSentence({ verb: walk.id, objectA: door.id });
// Wait for roomEntered event, then re-read state.

Incremental event reads

let cursor = 0;
setInterval(() => {
  const { events, cursor: c } = scumm1.eventsSince(cursor);
  cursor = c;
  for (const e of events) {
    if (e.type === "roomEntered") console.log("entered", e.room);
  }
}, 500);

Strategy

  1. Use scummN.read() as your primary orientation tool — inspect room objects, actors, verbs, and inventory to understand where you are and what you can do. Do not take screenshots for routine orientation.
  2. Use scummN.eventsSince(cursor) to efficiently catch up on what happened after an action — dialog text, room changes, cutscene starts/ends. Far cheaper than re-reading full state or taking screenshots.
  3. Plan before acting: read the full state, identify available objects and NPCs, form a goal, then execute.
  4. Collect everything you can. Most puzzles involve using inventory items — on objects, on other items, or giving them to NPCs. Pick up anything not nailed down.
  5. Build a mental map of room connections as you explore.
  6. Talk to NPCs to gather information.
  7. Do not touch scumm2 if you are agent 1, and vice versa. The log attributes every call to an agent id — crossing over corrupts the scoring data.

Rules

  1. Check scummN.actionsReady() before your first action.
  2. Check state.inputLocked before each action.
  3. Prefer doSentence over clickAt.
  4. Use events (not polling state) to detect action results.
  5. Avoid repeating failed actions.
  6. Stay on your assigned side.
